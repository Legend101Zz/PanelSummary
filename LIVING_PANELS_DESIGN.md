# Living Manga Panels — System Design

> Each manga panel becomes a **programmable canvas** driven by a DSL,
> interpreted by a rendering engine, and generated by an LLM.

---

## 1. DSL Design (v2.0)

### Core Philosophy
The DSL is **act-based**: each panel is a scene with temporal states.
The reader taps to advance between acts. Within each act, a timeline
auto-plays animations for layers.

### Top-Level Structure
```json
{
  "version": "2.0",
  "canvas": { "width": 800, "height": 600, "background": "#F2E8D5", "mood": "light" },
  "acts": [ ... ],
  "meta": {
    "panel_id": "identity-003",
    "chapter_index": 2,
    "content_type": "concept",
    "narrative_beat": "description",
    "duration_ms": 6000
  }
}
```

### Act Structure
Each act has:
- **layout** — how the act arranges its content (full, grid, or **cuts**)
- **layers** — visual elements (bg, sprites, text, effects)
- **cells** — sub-panels (for multi-panel layouts)
- **timeline** — keyframe animations
- **transition_in** — how this act enters (fade, cut, iris, slide)

### Layout Types

| Type | Description |
|------|-------------|
| `full` | Single panel |
| `split-h` | Left + right |
| `split-v` | Top + bottom |
| `grid-2x2` | 4 panels |
| **`cuts`** | **Recursive manga page cuts** |

### The Cut System (Key Innovation)

Inspired by comfyui_panels. A page starts as one panel.
Sequential cuts divide it:

```json
"layout": {
  "type": "cuts",
  "cuts": [
    { "direction": "h", "position": 0.55, "angle": 1.5, "target": 0 },
    { "direction": "v", "position": 0.65, "angle": -1, "target": 0 }
  ],
  "gap": 5,
  "borderWidth": 2.5
}
```

**Algorithm:**
1. Start with one full-page region `[0, 0, 1, 1]`
2. First cut (horizontal at 55%) → region 0 (top 55%) + region 1 (bottom 45%)
3. Second cut (vertical at 65% of region 0) → region 0 (top-left) + region 1 (top-right) + region 2 (bottom)
4. Result: 3 panel regions

**Cut Properties:**
- `direction`: `"h"` (horizontal) or `"v"` (vertical)
- `position`: 0.0–1.0, where to cut within target region
- `angle`: -4° to +4°, tilt for hand-ruled manga aesthetic
- `target`: which region index to cut (default 0)

### Layer Types

| Type | Purpose |
|------|--------|
| `background` | Gradients, patterns |
| `sprite` | Character silhouettes |
| `text` | Typography blocks |
| `speech_bubble` | Dialogue with tails |
| `effect` | Speed lines, screentone, SFX |
| `shape` | Geometric elements |
| `data_block` | Staggered lists/grids |

## 2. Rendering Engine Architecture

```
DSL JSON
  │
  ├─ LivingPanelEngine (React)
  │   ├─ Act state machine (tap to advance)
  │   ├─ PaperTexture (canvas noise)
  │   ├─ InkBorder (rough.js)
  │   │
  │   ├─ ActRenderer
  │   │   ├─ CutLayoutEngine (if type="cuts")
  │   │   │   ├─ computePanelRegions() → PanelRegion[]
  │   │   │   └─ Positioned divs with SVG ink borders
  │   │   │
  │   │   ├─ Grid Layout (if type="split-h" etc.)
  │   │   │
  │   │   └─ ActLayerStack
  │   │       ├─ LayerRenderers (per type)
  │   │       │   ├─ BackgroundLayer (CSS gradient + pattern SVG)
  │   │       │   ├─ SpriteLayer (ink silhouettes + name label)
  │   │       │   ├─ TextLayer (typewriter, manga fonts)
  │   │       │   ├─ SpeechBubbleLayer (hand-drawn bubbles)
  │   │       │   ├─ EffectLayer (speed lines, screentone)
  │   │       │   ├─ ShapeLayer (SVG primitives)
  │   │       │   └─ DataBlockLayer (staggered lists)
  │   │       │
  │   │       └─ AnimationRunner (Framer Motion)
  │   │           ├─ Timeline scheduler (setTimeout chain)
  │   │           ├─ Typewriter controller
  │   │           └─ Easing interpolation
  │   │
  │   └─ MangaInk (drawing primitives)
  │       ├─ PaperTexture (canvas noise)
  │       ├─ InkBorder (rough.js rectangles)
  │       ├─ CharacterSilhouette (SVG ink figures)
  │       ├─ SpeechBubble (hand-drawn paths)
  │       ├─ Screentone (dot matrix pattern)
  │       ├─ Crosshatch (line pattern)
  │       ├─ SpeedLines (radial/directional)
  │       ├─ InkSplash (splatter effect)
  │       └─ MangaSFX (rotated action text)
  │
  └─ Fonts: Dela Gothic One (display), Noto Serif JP (body),
            DM Sans (label), JetBrains Mono (data)
```

### Data Flow
```
DSL JSON → parse acts[] → ActStateMachine
  → on tap: advance act index
  → current act → ActRenderer
    → if cuts: computePanelRegions() → positioned divs
    → layers → LayerRenderers → DOM/SVG elements
    → timeline → setTimeout chain → Framer Motion animate()
```

## 3. LLM Generation Strategy

### Prompt Architecture
```
┌─────────────────────────────────────┐
│ SYSTEM PROMPT                       │
│ ├─ Role: Living Manga Panel Director│
│ ├─ Style guidance (per manga style) │
│ ├─ DSL v2.0 specification          │
│ │   ├─ Layer types + props          │
│ │   ├─ Layout types (incl. cuts)    │
│ │   ├─ Timeline/animation spec      │
│ │   └─ Positioning rules            │
│ ├─ Color palette (ink-on-paper)     │
│ └─ Critical rules (guardrails)      │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│ USER PROMPT (per panel)             │
│ ├─ Panel type + mood                │
│ ├─ Text content / dialogue          │
│ ├─ Character + expression           │
│ ├─ Manga Bible (characters, world)  │
│ └─ Chapter context                  │
└─────────────────────────────────────┘
```

### Guardrails
1. **JSON mode** — Force structured output
2. **Auto-fix** — `fix_common_dsl_issues()` patches:
   - Missing version → set to "2.0"
   - v1.0 flat format → wrap in acts
   - Missing layer IDs/props → defaults
   - Missing timeline fields → defaults
3. **Validation** — `validate_living_panel_dsl()` checks:
   - All acts have id, duration_ms, valid layout type
   - All layers have valid types
   - Cells have IDs
4. **Fallback** — If LLM fails entirely, generate from static panel data

### Positioning Rules (in prompt)
- Sprites at y: 60-75% (bottom of panel)
- Speech bubbles at y: 5-30% (top of panel)
- NEVER overlap text on sprites

## 5. Creative Ideas / New Medium

### Novel Panel Types
- **Branching dialogue** — Reader picks response, panel morphs
- **Data-driven panels** — Real stats animate in as manga infographics
- **Sound panels** — Audio waveform visualizations as manga SFX
- **Time-lapse panels** — Seasons/progress shown through animated layers
- **Emotion heatmaps** — Panel background shifts with story tension

### Unique Storytelling Mechanics
- **Unreliable narrator** — Text types then erases/rewrites itself
- **Memory panels** — Desaturated, sketchy, with paper aging effect
- **Future vision** — Clean lines gradually appearing over chaos
- **Inner monologue** — Layers of overlapping transparent thought text
- **The reveal** — Panel starts zoomed in, slowly pulls back

### What Makes This a New Medium
1. **Typography IS the art** — No images needed, text carries visual weight
2. **Temporal storytelling** — Acts give panels a time dimension
3. **Reader-controlled pacing** — You decide when the next beat hits
4. **Infinite cut complexity** — Nested cuts = infinite panel layouts
5. **Code-rendered aesthetics** — Screentone, crosshatch, speed lines
   are procedural, not drawn
6. **LLM as director** — The AI chooses timing, layout, and emphasis
